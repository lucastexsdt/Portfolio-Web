Virtual Server Vulnerability Assessment

Security Analysis Nmap OpenVAS

Project Overview

This project involved conducting a comprehensive vulnerability assessment on two Linux virtual machines in a controlled lab environment. The objective was to identify security weaknesses, document findings with industry-standard CVE references, and develop a prioritized remediation strategy based on risk severity.

My Role & Involvement

Security Analyst

I independently led this vulnerability assessment project from planning through execution and documentation. My responsibilities included configuring the lab environment, executing scans using industry-standard tools, analyzing results, mapping vulnerabilities to CVE identifiers, assessing risk levels, and producing comprehensive security documentation with actionable remediation recommendations.

Work Process & Methodology

1. Environment Setup & Planning

Configured two Linux virtual machines in an isolated lab environment. Reviewed project scope and defined assessment objectives aligned with industry best practices.

2. Network Reconnaissance (Nmap)

Performed comprehensive network scanning using Nmap to identify live hosts, open ports, running services, and service versions. Documented all exposed services and potential attack vectors.

3. Vulnerability Scanning (OpenVAS)

Executed automated vulnerability scans using OpenVAS against both target systems. Configured scan profiles to detect configuration weaknesses, outdated software versions, and known security vulnerabilities.

4. Results Analysis & CVE Mapping

Analyzed scan results to identify 18 distinct vulnerabilities. Cross-referenced findings with the CVE database to assign standardized identifiers and severity ratings (Critical, High, Medium, Low).

5. Risk Assessment & Prioritization

Evaluated each vulnerability's potential business impact considering exploitability, data sensitivity, and system criticality. Prioritized remediation actions based on risk severity.

6. Documentation & Reporting

Produced detailed security assessment report including executive summary, technical findings, CVE mappings, risk ratings, and prioritized mitigation recommendations for each identified vulnerability.

Network Forensics & Ransomware Analysis

Wireshark Sguil Kibana Incident Response

Project Overview

Investigated a security breach at "Lion Health" involving the CryptoWall ransomware. Using packet analysis and SIEM tools, I reconstructed the full attack chain—from the initial drive-by download via the "Angle Exploit Kit" to the final C2 communication.

My Role & Involvement

Incident Response Analyst

I served as the lead analyst for this incident investigation, responsible for examining all available evidence including network packet captures, security alerts, and log data. I performed deep packet analysis, correlated events across multiple data sources, identified malicious activity patterns, and produced a comprehensive incident report with technical findings and strategic recommendations.

Work Process & Investigation Steps

1. Initial Triage & Evidence Collection

Gathered all available evidence including PCAP files, security alerts from Sguil, and log data accessible through Kibana. Established investigation scope and documented initial observations.

2. Network Traffic Analysis (Wireshark)

Performed detailed packet-level analysis using Wireshark to examine network communications, identify suspicious traffic patterns, extract file transfers, and analyze protocol behaviors. Applied display filters to isolate relevant traffic flows.

3. Alert Correlation (Sguil)

Reviewed and validated security alerts generated by intrusion detection systems through Sguil interface. Correlated alerts with PCAP evidence to distinguish true positives from false positives and establish event relationships.

4. Log Analysis & Visualization (Kibana)

Utilized Kibana to search, filter, and visualize log data across the investigation timeframe. Created custom visualizations to identify anomalies, track attacker movements, and support timeline reconstruction.

5. IOC Identification & Timeline Construction

Identified key indicators of compromise including malicious IP addresses, suspicious file hashes, unusual processes, and command-and-control communications. Constructed a detailed incident timeline mapping attacker actions chronologically.

6. Reporting & Recommendations

Produced formal incident report documenting investigation methodology, technical findings, IOCs, attack timeline, business impact assessment, and evidence-based containment and prevention recommendations to strengthen security posture.

Web Development Flask SQLite

Project Overview

This full-stack web application project involved designing and developing a content management system for stories and memories with comprehensive CRUD (Create, Read, Update, Delete) operations. The application features user interaction capabilities, database management, input validation, secure file upload functionality, and proper error handling to ensure both functionality and security.

My Role & Involvement

Full Stack Developer — Story & Memory Management System

I architected and built the complete Story & Memory Management feature. On the Backend (Flask & SQLite), I designed the database schema, implemented CRUD routes, and created the "For You" sorting algorithm. On the Frontend, I built the responsive interface using Bootstrap 5, integrated the "What's New" carousel, and implemented real-time user interactions.

Development Process & Implementation

1. Requirements & Planning

Defined application requirements for story/memory management system including CRUD operations, user interactions (likes, comments, bookmarks), and content moderation features. Designed database schema and planned application architecture.

2. Database Design & Setup (SQLite)

Designed relational database schema to support stories, comments, likes, views, bookmarks, and report functionalities. Implemented proper relationships, constraints, and indexes for optimal performance and data integrity.

3. Backend Development (Flask)

Developed Flask backend routes to support CRUD for blog posts and user interactions. Integrated SQLite for data storage and handled create/update/delete workflows reliably. Implemented server-side validation and error handling to ensure clean submissions.

4. Security Implementation

Implemented server-side input validation for titles and content to reduce invalid or unsafe submissions. Added file upload validation with size limits and allowed file type checks. Used parameterized database queries to improve data safety and integrity.

5. Frontend Development & UX

Built responsive pages using HTML/CSS and Bootstrap. Integrated flash messages and toast notifications to guide users during errors and submissions. Added basic client-side checks to improve usability alongside server-side validation.

6. Testing & Documentation

Tested key workflows (publish, update, image upload, validation errors) to ensure features work correctly. Verified database operations for posts and interactions (comments, likes, bookmarks, reports). Documented feature flow and validation rules with screenshots and notes for the project report.
